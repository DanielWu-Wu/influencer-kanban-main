## 1. 项目整体目标

本项目是面向跨境电商海外红人推广专员的桌面 CRM 工作台，重点服务 YouTube 红人合作。目标是把频道发现、红人建档、Gmail 开发与回复、AI 辅助起草、飞书记录、产品资料、合作履约、视频发布和后续复盘集中到一套清晰、可控的流程中。

核心边界：AI 负责分析、建议、翻译、起草和预览；Gmail 发送、Gmail 草稿、飞书写回等外部动作必须由用户明确触发。当前合作项目中心只读飞书，不自动修改外部数据。

## 2. 技术栈和核心架构

- Next.js 16.1.1 App Router、React 19.2、TypeScript 5。
- Tailwind CSS 4、shadcn/ui、Radix UI、lucide-react。
- Next.js Route Handlers 位于 `src/app/api/**`，承接 Gmail、飞书、YouTube、AI、鉴权等服务端能力。
- Supabase 用于登录、云端设置和用户私密配置；部分业务状态仍使用本地存储。
- 外部集成：Gmail API、飞书多维表格 API、YouTube Data API、OpenAI-compatible AI API。
- 主页面壳和导航在 `src/app/page.tsx`；业务 UI 在 `src/components`；字段映射、缓存和业务规则在 `src/lib`。
- 飞书字段必须通过用户保存的语义映射读取或写入，不能依赖固定中文列名。

## 6. 合作项目重要业务逻辑

### 阶段判断

按已到达的最高节点判断，未来日期不算已到达：

1. 有实际上线日期：视频已发布。
2. 有视频拍摄完成时间：视频拍摄完成。
3. 有到货时间：到货当天显示“已到货”，下一天起自动进入“视频拍摄中”。
4. 有发货时间：运输中。
5. 明确需要样品且未发货：待发货。
6. 明确无需样品：合作日期当天为“合作已确认”，之后自动进入“视频拍摄中”。
7. 其他有效记录：合作已确认。

拍摄开始时间不单独写入飞书：寄样项目从到货时间推断；无需寄样项目从合作日期推断。推断节点在时间线中标注“自动推断”。

### 风险和下一步

- 预计发布日期已过但无实际上线日期：标记逾期。
- 有视频链接但无实际上线日期：提示补录日期，不直接判定已发布。
- 已发货但未勾选物流信息已告知：提示告知红人。
- 日期顺序矛盾、已到货但缺发货时间、拍摄完成但缺到货时间：显示异常或缺失提示。
- 下一步建议由当前阶段、物流单号、告知状态和视频链接共同计算。

### 数据和安全边界

- 页面只调用现有飞书记录读取接口，不写回飞书，不新增数据库表。
- 看板不能拖拽，阶段必须由日期和业务字段计算，避免形成第二套人工状态。
- 手动刷新使用 `{ force: true }` 绕过缓存；普通进入页面复用 60 秒缓存。
- 缺少配置、授权失败、读取失败或字段映射缺失时必须给出明确提示，不能静默显示空白。

- “已到货”目前是到货当天的短暂阶段，次日自动进入拍摄中。这是当前确认的自动规则，若业务需要人工确认拍摄开始，应新增明确字段后再调整。

- 全仓库 ESLint 当前有 9 条历史 warning：`email-detail.tsx` 有 1 条 `<img>` 性能提示，
